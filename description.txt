Findr AI
AI Community + Web Search Agent

এটা হবে এমন এক agent যেটা মানুষকে Facebook group/community post + Google/web trusted source থেকে accurate useful information খুঁজে দিবে। শুধু search result list না; agent বুঝবে user আসলে কী চায়, source খুঁজবে, evidence verify করবে, rank করবে, তারপর answer দিবে।

Hackathon-এর জন্য এটা fit কারণ challenge চায় agent যেন শুধু answer না দেয়, real tool দিয়ে কাজ করে; Google Cloud Agent Builder/Gemini + partner MCP integration লাগে; আর judging criteria-তে technological implementation, design, impact, idea quality আছে।

1. Project problem

Facebook group/community group-এ অনেক useful post থাকে:

internship
research idea
job circular
scholarship
admission

কিন্তু problem:

Facebook search weak
keyword না মিললে post পাওয়া যায় না

So তোমার project solve করবে:

“আমি যা চাই, সেই intent বুঝে old group/community post + trusted web source থেকে accurate answer খুঁজে দাও, evidence সহ।”

2. One-line Devpost pitch

Findr AI is an Elastic MCP-powered Gemini search agent that turns noisy Facebook/community posts and trusted web sources into accurate, evidence-grounded answers for internships, research ideas, jobs, scholarships, coding help, and local student needs.

D. Elasticsearch

Elastic is the core search engine.

It stores:

Facebook/community posts
comments
web pages
official links
saved resources
PDF/CSV/bookmarks

It searches using:

keyword search
semantic/vector search
metadata filter
hybrid ranking
freshness boost
source trust score
duplicate removal

Why Elastic makes it strong:

Normal search = word match
Elastic hybrid search = word + meaning + filters + ranking

Example:

User asks:

"AI cybersecurity research idea"

Post says:

"ML-based intrusion detection project"

Keyword search may miss it. Semantic/vector search can match the meaning.

E. Elastic MCP Server

This is the hackathon partner superpower.

Instead of Gemini directly calling your backend only, Gemini will use Elastic MCP tools.

Flow:

Gemini Agent
→ calls Elastic MCP tool
→ Elastic MCP queries Elasticsearch
→ returns ranked evidence
→ Gemini verifies and explains

This matters because hackathon requires meaningful partner MCP integration. The challenge text says you must build an agent with Gemini/Google Cloud Agent Builder and integrate a partner MCP server.

F. Gemini / Google Agent Builder

Gemini is not just answering. It is planning.

Gemini agent does:

understand user intent
decide sources
call Elastic MCP
call web search
compare evidence
remove weak result
write final answer
suggest next action

Agent Builder is useful because the hackathon asks for an agent that can reason, plan, and execute tasks under user oversight, not just a chatbot.

This is what makes answer accurate.

It checks:

Is it recent?
Is it official or random?
Does snippet prove the answer?
Is source duplicate?
Is the deadline expired?
Is the post only discussion or real opportunity?

Output:

I. Trust Score Engine

Each result gets score.

Example formula:

Final Score =
semantic relevance
+ keyword match
+ freshness
+ official source boost
+ useful comment signal
+ deadline detected
- duplicate penalty
- outdated post penalty
- vague post penalty

For internship:

official source + recent + deadline + apply link = high score
old vague post = low score

For research idea:

clear idea + methodology + paper/source link = high score
random one-line comment = low score

10. Safe Facebook strategy

Do not make it like old scraper projects.

Existing Facebook scraper projects are mostly data collection tools; the uploaded GitHub README says that project scrapes Facebook public pages, supports group parameters, comments, CSV/JSON, and also warns group scraping may miss fields, not work on private groups, and too much scraping can temporarily ban IP.

Your project should say clearly:

We do not mass scrape Facebook.
We do not bypass login.
We do not auto-scroll.
We only index visible content after user clicks the button.
Demo uses sample data or user-owned/approved data.

This makes your project safer and more professional.
